npm 命令

初始化 npm 项目的时候可以使用 npm init

基本使用方式为 npm install <name> [--save] # --save 参数用以将安装的模块加入到package.json中

需要安装为命令的时候可以使用 npm install --global <name>

如果需要通过package.json来安装的话（在当前目录下有package.json），可以直接执行npm install

更多的使用方式，可以参考 npm help install，或者更通用的 npm help

bower 命令

初始化 bower 项目的时候可以使用 bower init

基本使用方式为 bower install <name> [--save] # --save 参数用以将安装的组件加入到bower.json中

如果需要通过bower.json来安装的话（在当前目录下有bower.json），可以直接执行 bower install

更多的使用方式，可以参考 bower help install，或者更通用的 bower help

express 命令

这个命令本身并不是 express 模块的一部分

它其实是 express-generator 模块的一部分

我们可以通过 npm install -g express-generator 来安装它

为什么我们要使用它呢，因为它可以替我们生成基本的 express 项目

一个 express 项目本身是一个 npm 项目，可以认为是包含了 express 这个依赖的 npm 项目就算是它了

nodemon 命令

这个命令是推荐安装的，实现自动重启服务器

自动重启服务器可以使用grunt配合插件使用，但是，这个对于新手来说更加容易接受

这个命令的使用方式就是在项目根目录中，使用 nodemon 启动该项目

详见第一个例子

一个例子

  1. npm install -g express-generator # 先安装 express-generator
  2. express blog # 创建一个express（npm）项目，名字为blogk
  3. cd blog; npm install # 进入项目目录，安装所需要的依赖（这里就包括express模块）
  4. npm start # 启动项目，也可以执行 node bin/www 来启动项目
  5. nodemon bin/www # 这个和上一点可以互相替代，使用这个命令可以在服务器代码发生变化的时候自动重启

另一个例子

  1. mkdir app; cd app # 创建一个名为 app 的目录
  2. bower init # 初始化 bower 项目
  3. bower install --save angular # 安装 angular 依赖，并将其保存到 bower.json 中
  4. bower install --save bootstrap # 安装 bootstrap 依赖，并将其保存到 bower.json 中
  5. 在这之后，我们就可以在需要引用angular 和 bootstrap 的网页中加入相应的引用代码

再来一个例子：当这个项目来自github，使用 npm 管理后台，bower 管理前台

  1. cd <project>; npm install # 安装后台项目依赖
  2. cd <client>; bower install; cd - # 安装项目的前端依赖，这里的client是说静态文件所在的位置
